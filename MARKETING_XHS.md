## 版本一：硬核会务版

**标题建议：**  
做了 5 场 MUN 之后，我干脆把“主席团后台”写成了一个完整系统…

**正文：**

第一年当 MUN 主席团的时候，我最崩溃的不是代表吵架，而是——  
Excel 开十几个、群消息 99+、文书传来传去，全靠人肉记忆。

于是我用一整个学期，把这些流程全部抽象成了一个系统，现在已经跑完一整套大会，准备挂出来给需要的主办方使用。

📌 这个系统能做什么？

### 1⃣ 账号 & 权限管理

- 支持 `admin / dais / delegate / observer` 四种角色  
- 每个用户可以单独配置权限（比如“只能管理某一个会场”）  
- 支持 Excel 一键导入 / 导出参会名单，批量创建账号、重置密码

### 2⃣ 会场 & 代表管理

- 一键配置多个会场：名称、代码、地点、主席团、容量、时间设置  
- 代表分配到具体会场 + 国家，可以设置是否有否决权  
- 支持 CSV 导入代表，快速生成一整场 MUN 的代表名单  

### 3⃣ 会议进行中：电子大屏 + 动议 + 发言名单

- 每个会场都有独立的**电子大屏**：当前议题、发言名单、出席情况、动议历史一目了然  
- 主席团可以实时点名，系统自动计算出席人数与通过门槛  
- 发言名单支持队列管理：等待中 / 发言中 / 已结束

### 4⃣ 文件 & 文书流转

- 代表在线提交决议草案 / 工作文件  
- 主席团在后台审核：通过 / 驳回 + 反馈意见  
- 审核通过后可一键发布为“正式文件”，所有代表在线查阅

### 5⃣ 危机 & 反馈 + 日志审计

- 危机玩法：主席团发布危机，限定会场看到；代表提交行动方案和资源请求  
- 所有关键操作有日志记录，方便赛后复盘、责任追踪  

🧱 技术上：

- 前端：Vue3 + Vite + Tailwind CSS + daisyUI，样式统一，响应式适配  
- 后端：PHP8 + MySQL，接口全部有 OpenAPI 文档，方便二次开发对接  
- 支持 Docker 一键部署到服务器（配好域名和 HTTPS 就能直接用）

## 版本二：产品介绍版（强调省心）

**标题建议：**  
把一整场模拟联合国大会塞进一个后台系统，是种什么体验？

**正文：**

开过 MUN 的人都懂：  
真正累人的不是开会，而是**报名表、文书、动议、点名、发言名单**这一堆琐碎流程。

我做了一个完整的「MUN 大会后台系统」，现在已经能从**大会前期准备 ➝ 会议进行中 ➝ 会后归档**全程接住。

🌟 一句话总结：

> 「你只管当主席，我帮你管所有数据。」

---

🔐 **账号与权限**

- 统一登录、修改密码  
- 支持管理员 / 主席团 / 代表 / 观察员多种身份  
- 每个用户可以单独开关权限（比如：只能看自己会场）

🏛 **会场 & 代表**

- 多会场并行：一场大会 N 个委员会都能配置  
- 为每个代表绑定会场 + 国家 + 否决权  
- Excel 导入导出，适合从报名表直接转成系统数据  

🧑‍⚖️ **开会过程：大屏 + 动议 + 发言名单**

- 每个会场对应一块**电子大屏**（网页即可投屏）：  
  - 议题、当前阶段、发言名单  
  - 出席代表计数 & 通过门槛  
  - 近期动议的通过 / 否决记录  
- 主席团只需要在后台点击：点名、通过 / 否决动议、调整发言顺序，系统自动更新大屏  

📄 **文书：提交 ➝ 审核 ➝ 发布**

- 代表在自己的账号里上传文书  
- 主席团直接在后台审批，写好反馈意见  
- 通过后成为「公开文件」，所有人都可以在线查看 & 下载  

⚡ **危机玩法 & 操作日志**

- 主席团可以推送 crisis 给指定会场  
- 代表在系统里填写反馈，提交行动方案  
- 所有数据库操作都会记录日志，方便赛后复盘  

---

🧱 **基础设施**

- 前端：Vue3 + Tailwind + daisyUI，支持定制主题  
- 后端：PHP + MySQL，接口有完整开放文档  
- 支持 Docker 一键部署，方便放到你们学校 / 机构自己的服务器上

## 版本三：故事感版（强调真实痛点）

**标题建议：**  
当我们把 MUN 大会的混乱，变成了一块安静的后台页面

**正文：**

以前开 MUN，大概是这样的画风：

- 主席团手机、电脑、平板一起上阵  
- 文书在群文件、邮箱、U 盘里到处飞  
- 每天都有人问：「这动议刚刚投票结果是多少来着？」  
- Excel 表保存出一堆「最终版 v6-final-真的最终.xlsx」

后来我们决定，干脆自己把这些东西做成一个系统。

这一版系统已经跑过完整的模拟联合国大会，现在可以稳定处理：

- 多会场并行  
- 上百名代表账号  
- 各种动议、危机、文件流转  

我简单介绍几个，对主席团最友好的功能 👇

---

1⃣ **所有代表都“长在”一个后台里**

- 名单导入一次搞定：姓名、邮箱、会场、国家、权限  
- 重置密码、锁定账号、修改信息全在一个界面里解决  

2⃣ **会议进行中：一切操作只要“点一下”**

- 点名：一次提交，系统自动给你算出席人数 & 多数门槛  
- 发言名单：增加 / 删除 / 调整顺序，全是按钮操作  
- 动议：记录发起人、内容、投票结果，自动写入历史  

3⃣ **文书 & 危机，不再散落在各种群聊里**

- 文书：代表上传后，主席团在后台一眼看到所有待审文件  
- 危机：可以指定推送给某个会场，代表在线填回应对方案  
- 所有操作留痕，方便你会后写总结 / 复盘  

---

技术上，我们用的是：

- Vue3 + Tailwind + daisyUI 做前端  
- PHP + MySQL 做后端，全部 API 有文档  
- 支持 Docker，一键部署到云服务器
